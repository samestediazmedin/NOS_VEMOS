Se cerro una ronda completa de mejoras en frontend y flujo operativo para dejar el proyecto listo para entrega tecnica.

## Cambios implementados

1. **Calidad tecnica y verificacion**
   - Se ejecuto verificacion de seguridad del repositorio (`verify-security.ps1`): build y pruebas en verde.
   - Se instalo dependencias del frontend y se valido build de Vite.

2. **Ajustes de documentacion y limpieza base**
   - Se corrigio el comando repetido en `README.md` para arranque de `Usuarios`.
   - Se actualizo catalogo de microservicios con estado real de `Auditoria`.
   - Se limpio ruido de entorno local (`.gitignore` para `node_modules`).

3. **Rediseño UI/UX del frontend movil**
   - Se aplico sistema visual completo (tokens, estados de botones, contrastes, layouts responsive).
   - Se mejoro login visual y experiencia general de navegacion.
   - Se reforzaron estados vacios, feedback y jerarquia visual en pantallas principales.

4. **Control de acceso y roles**
   - Se simplifico a un solo rol operativo: `Admin`.
   - Se retiro seleccion de roles no requeridos en login.
   - Se fijo validacion de credenciales de administrador para acceso.

5. **Gestion de usuarios en proceso unico**
   - Se creo modulo `Gestion de usuarios` con alta integrada.
   - El alta ahora exige consentimiento + 3 capturas biometricas validas (`frontal`, `izquierda`, `derecha`).
   - Se habilito recaptura por angulo y bloqueo de alta si no cumple el flujo.
   - Se guarda estado biometrico del usuario y metadata local de enrolamiento.

6. **Navegacion y flujo de inicio**
   - Se retiro modulo separado `Captura Asistida` del menu.
   - La aplicacion vuelve a `Dashboard` al iniciar sesion y al cerrar sesion.
   - Se actualizo etiqueta de menu a `Gestion de usuarios`.

7. **Historial operativo**
   - Se agrego en historial una tabla por usuario con hora de ingreso al sistema.
   - Se mantuvo historial de reconocimientos y enrolamientos para trazabilidad.
